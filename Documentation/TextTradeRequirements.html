Introduction

Our project is an application named Text Trade that is basically a textbook marketplace and allows Austin Peay students to come together and buy or sell their used textbooks. Our team plans to keep the application simple and easy to use. There will be two essential GUIs: one for users, and one for moderators. Each user has to register an account in order to start trading on Text Trade. The user then can create a list of books he wants to buy and/or sells. Each textbook entry must include all the necessary information such as title, author, edition, ISBN, associated course(s) that the textbook is used for, and book condition. A user can search for a book in the marketplace, find potential buyers, communicate with other traders, and report misdeeds to moderator. To keep things neat, we plan to implement the search algorithm based on ISBN numbers and course numbers. User Requirements Definition

  • Buyer

    1. Name: view_Listings
      Type: Functional
      Summary: Buyers must be able to view listings

    2. Name: manage_Schedule
      Type: Functional
      Summary: Buyers shoudl be able to add and drop classes from their schedule

  • Seller

    1. Name: add_Listing_Seller
      Type: Functional
      Summary: Sellers shall be able to create new listing. For each book, the following fields should be filled out: Book title, Author, Edition, ISBN, Class used for, Semester used for, Condition, Price

    2. Name: remove_Listing_Seller
      Type: Functional
      Summary: Sellers shall be able to remove their own listings.

    3. Name: edit_Listing_Seller
      Type: Functional
      Summary: Sellers shall be able to edit their own listings info.

  • Moderator

    1. Name: remove_user_Mod
      Type: Functional
      Summary: Moderators shall be able to ban users who commit foul play

    2. Name: remove_Listing_Mod
      Type: Functional
      Summary: Moderators shall be able to remove listings deemed inappropriate

  • Name: search_manually
    Type: Functional
    Summary: Users shall search for desired textbooks by manually typing string into search bar

  • Name: create_account
    Type: Functional
    Summary: Users shall create an account to participate in Text Trade marketplace

  • Name: edit_profile
    Type: Functional
    Summary: Users should complete their profile page, filling out fields such as full name, DOB, phone number, campus email address, major, concentration, etc.

  • Name: report_user
    Type: Functional
    Summary: Users should be able to report inapproriate activities to Moderators.
