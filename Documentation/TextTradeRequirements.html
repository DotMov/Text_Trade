Introduction

Our project is an application named Text Trade that is basically a textbook marketplace and allows Austin Peay students to come together and buy or sell their used textbooks. Our team plans to keep the application simple and easy to use. There will be two essential GUIs: one for users, and one for moderators. Each user has to register an account in order to start trading on Text Trade. The user then can create a list of books he wants to buy and/or sells. Each textbook entry must include all the necessary information such as title, author, edition, ISBN, associated course(s) that the textbook is used for, and book condition. A user can search for a book in the marketplace, find potential buyers, communicate with other traders, and report misdeeds to moderator. To keep things neat, we plan to implement the search algorithm based on ISBN numbers and course numbers. As for the moderator’s GUI, there are additional features that allow a moderator to manage the database, ban a disruptive user, or remove postings that sell non-textbook merchandises, etc.

User Requirements Definition

  • User/Trader

    1. Name: search_marketplace
      Type: Functional
      Summary: Users shall be able to search Text Trade marketplace for textbooks using book title, ISBN, associated courses, etc.

    2. Name: view_Listings
      Type: Functional
      Summary: Users shall be able to view listings/search results

    3. Name: control_search_results
      Type: Functional
      Summary: Users shall be able to filter and sort search results based on given criteria

    4. Name: manage_Schedule
      Type: Functional
      Summary: Users shall be able to add and drop classes from their schedule

    5. Name: manipulate_Listings
      Type: Functional
      Summary: Users shall be able to add, remove, or edit their own listings

    6. New - Name: communicate_internally
      Type: Functional
      Summary: Users shall be able to communicate with other traders/moderators via Text Trade internal messaging system

    7. Name: report_misdeed
      Type: Functional
      Summary: Users shall be able to report to moderators inappropriate listings or disruptive users

  • Moderator

    1. Name: moderator_Powers
      Type: Functional
      Summary: Moderators shall be able to manipulate the database, communicate with other users, remove listings, view users' protected info, issue warnings, ban/block users, and view system stats

    2. New - Name: moderator_restriction
      Type: Constraint
      Summary: Moderators shall not be able to post their own listings

  • System

    1. New - Name: store_data
      Type: Functional
      Summary: System shall retain data such as user info, listings, internal messages, system stats, etc.

    2. New - Name: database_access_restriction
      Type: Constraint
      Summary: Regular user's access to database is restricted to certain domain. Moderator has greater access and control over databse

    3. Name: search_database
      Type: Functional
      Summary: Application shall be able to search through database table for target data as requested

    4. Name: create_Account
      Type: Functional
      Summary: Users shall create accounts to access Text Trade marketplace

    5. New - Name: distinct_GUI
      Type: Users and Moderators shall have separate interfaces
      Summary:

    6. New - Name: email_notification
      Type: Functional
      Summary: Users should be notified by email when transactions are approved, listings responded to, etc.

System Requirements Specifications

  • User/Trader

    1. Name: edit_profile
      Type: Functional
      Summary: Users should complete their profile page, filling out fields such as full name, DOB, major, concentration, etc.

    2. Name: report_user
      Type: Functional
      Summary: As user views another user’s profile page, he has the option to report that user to moderators. (maybe include a button with flag icon on the user profile page)

    3. Name: create_class_schedule
      Type: Functional
      Summary: User can create a list of his currently registered classes for which he needs to buy textbooks. He may label the schedule (Ex: Spring 2017 schedule) and add each class by filling out “Subject” and “Level” fields (Ex: #1 ENGL 1010, #2 MATH 1920, #3 HIST 2030)

    4. Name: edit_schedule
      Type: Functional
      Summary: User can edit his own schedule; remove, change, or add class

    5. Name: save_listing
      Type: Functional
      Summary: Users shall be able to save listings of interest into their Watch list

    6. Name: create_Listing_User
      Type: Functional
      Summary: User who wants to sell a textbook may create a listing by filling the following fields and publish it.
      • Book title (required field)
      • Author (required field)
      • Edition (required field)
      • ISBN (optional)
      • Associated APSU course (Subject: ____, Level: ____ ; ex: Subject: CSCI, Level: 4600) (optional)
      • When the book was used for that class (Semester: ____, Year: ____; ex: Semester: Spring, Year: 2017) (optional)
      • Book condition (required field)
      • Asking price or best offer (required)
      • Picture of the textbook (optional)
      • Additional note/description – text box (optional)

    7. Name: remove_Listing_User
      Type: Functional
      Summary: User may remove their own listing before the expiration date

    8. Name: edit_Listing_User
      Type: Functional
      Summary: User may make amends to their published listing

    9. New - Name: send_message
      Type: Functional
      Summary: Text Trade user/moderator can send messages to other Text Trade users/moderators

    10. New - Name: message_length_constraint
      Type: Constraint
      Summary: Each message can contain 1000 characters max

    11. New - Name: receive_message
      Type: Functional
      Summary: User/moderator can receive messages from other users/moderators

    12. New - Name: reply_message
      Type: functional
      Summary: User/moderator can reply to messages from other users/moderators

    13. New - Name: delete_message
      Type: Functional
      Summary: User/moderator can delete messages from personal inbox

    14. New - Name: listing_life_constraint
      Type: constraint
      Summary: All listings have a life span of 30 days. A listing is expired after 30 days

    15. New - Name: renew_listing_life
      Type: Functional
      Summary: Anytime, the user can choose to renew an active listing life (of his or hers) for another 30 days

    16. New - Name: search_manually
      Type: Functional
      Summary: User/moderator can manually type strings into search box to search for an item/textbook

    17. New - Name: search_by_course
      Type: functional Summary: Uer/moderator can search for listings by associated course by picking “Search by course” option, then field out “Subject” and “Level” fields (Ex: MATH 1910)

    18. New - Name: search_by_isbn
      Type: Functional
      Summary: User/moderator can search for listings using isbn by picking “Search by ISBN” option and entering ISBN number of the desired book.

    19. New - Name: search_from_schedule
      Type: Functional
      Summary: User has the option to let the system automatically generates search results based on each schedule input/class (results are matched by listing’s “Associated course” field)

    20. New - Name: delete_saved_listing
      Type: Functional
      Summary: User may remove active listings from Watch list as desired

    21. New - Name: inquiry_seller
      Type: Functional
      Summary: As user views a listing, he has the option to contact the seller via internal messaging system. (maybe include a button labeled “Contact Seller” on the listing page)

    22. New - Name: report_listing
      Type: Functional
      Summary: User may report an inappropriate listing to moderators. (maybe include a button labeled “Report this Listing” on the listing page)

    23. New - Name: report_user
      Type: Functional
      Summary: As user views another user’s profile page, he has the option to report that user to moderators. (maybe include a button with flag icon on the user profile page)

  • Moderator

    1. Name: remove_Listing_Mod
      Type: Functional
      Summary: Moderator may remove listings deemed inappropriate

    2. Name: ban_User_Mod
      Type: Functional
      Summary: Moderator may ban a disruptive user/trader temporarily or eternally. Ban duration can be determined flexibly by the moderator

    3. New - Name: ban_user_mod_constraint
      Type: Constraint
      Summary: moderator can only ban regular users/traders, he cannot ban another moderator

    4. New - Name: issue_warning_mod
      Type: Functional
      Summary: Moderator may issue warnings to user(s) upon violation

    5. New - Name: search_user_mod
      Type: Functional
      Summary: Moderator can search for a user in the system to view his/her information

    6. New - Name: view_user_info_mod
      Type: Functional
      Summary: Moderator can view a user's information, including registered APSU email address, password, and count of warnings

    7. New - Name: view_system_stats_mod
      Type: Functional
      Summary: Moderator can view certain system statistics such as # of new accounts created last month, # of users total, # of listings created last month, # of active listings, etc.

  • System

    1. Name: create_Account
      Type: Functional
      Summary: Users must create accounts to access Text Trade marketplace

    2. New - Name: register_email_constraint
      Type: Constraint
      Summary: User shall register with APSU email address

    3. New - Name: username_constraint
      Type: Constraint
      Summary: User shall pick a unique username that contains at least 5 characters that are either alphabet letters or numerical

    4. Name: sort_results
      Type: Functional
      Summary: Results can be sorted based on criteria such as:
      • Time: Newest listing first
      • Time: Ending soonest first
      • Price: Cheapest first
      • Price: Highest first

    5. Name: filter_results
      Type: Functional
      Summary: Results can be filtered based on:
      • Price threshold: less than X $ or greater than X $
      • Condition: New or Used (Like New, Great, Good, Acceptable)

    6. Name: listing_Table
      Type: Functional
      Summary: Database must hold a table of the listing

    7. Name: personal_Information
      Type: Functional
      Summary: Users should enter in personal Information on first time login

    8. New - Name: email_notification
      Type: Functional
      Summary: Users should be notified by email when transactions are approved, listings responded to, etc.

    9. Name: login_Storage
      Type: Risk
      Summary: Login information must be encrypted

    10. Name: password_Requirements
      Type: Constraint
      Summary: Passwords must contain atleast 1 uppercase, 1 lowercase, be atleast 6 characters long, and 1 number

    11. New - Name: get_report_reason
      Type: Functional
      Summary: As user clicks the report button, he is prompted to enter the reason for reporting a listing or another user into a text box

    12. New - Name: report_constraint
      Type: Constraint
      Summary: Without entering a reason, the user cannot send report.

    13. New - Name: remove_expired_listing_system
      Type: Functional
      Summary: An expired listing will be removed automatically by the system

    14. New - Name: display_listing_remaining_life
      Type: Functional
      Summary: Display how many days are left until expiration for each listing

    15. New - Name: store_message
      Type: Functional
      Summary: All received and sent messages shall be stored in an inbox

    16. New - Name: count_warning
      Type: Functional
      Summary: System keeps track of how many warnings each user has received. This information is visible only to moderators.
